# JARVIS UI 重设计 PRD

## 文档信息

| 字段 | 内容 |
|------|------|
| 产品名称 | JARVIS — 智能售前准备助手 |
| 文档版本 | v1.0 |
| 创建日期 | 2026-06-24 |
| 重设计范围 | 渐进式重做（保留 Streamlit 框架，逐页面重写 UI） |
| 优先级页面 | 首页 → 智能 Prep → 模拟训练 → 知识库 |

---

## 1. 背景与动机

### 1.1 当前问题

JARVIS 当前 UI 基于 Streamlit 默认样式 + 少量自定义 CSS，存在以下问题：

1. **专业感不足** — 使用 emoji 作为功能图标（🛡️🎭📚🔍），在企业级售前场景下显得不专业
2. **信息架构冗余** — 首页"核心功能"和"快速开始"展示同一组功能，信息重复
3. **侧边栏导航重复** — Streamlit 自动生成的页面导航 + 自定义"快速导航"按钮，同一功能出现两次
4. **视觉层级扁平** — 生成结果用 6 个权重相同的 expander 展示，缺少核心/补充的区分
5. **交互反馈不足** — LLM 生成过程只有 spinner，10-30 秒等待期间用户无法感知进度

### 1.2 重设计目标

将 JARVIS 从"Streamlit 原型"提升为"品牌质感产品"，让用户（企业售前/安全决策者）感知到这是一个**专业的、可信赖的**工具，而非简单的 Demo 或小玩具。

---

## 2. 设计原则

| 原则 | 说明 |
|------|------|
| **专业优先** | 所有视觉元素必须传递"可信赖"的信号；禁止使用 emoji 作为功能性图标 |
| **内容优先** | UI 服务于信息传达，不为装饰而装饰；减少视觉噪音，让核心信息突出 |
| **渐进披露** | 核心信息直接可见，补充信息折叠/分区；用户不需要额外操作就能看到最重要的内容 |
| **一致性** | 跨页面保持统一的色彩、图标、间距、交互模式；建立组件库复用 |

---

## 3. 视觉设计规范

### 3.1 风格定位

**品牌质感风**（参考 Stripe / Anthropic / Palantir）

- 有品牌色体系，不是纯白底灰字
- 精致卡片 + 微动效，介于"工具"和"品牌"之间
- 信息密度适中，不过度留白也不拥挤

### 3.2 色彩系统

**双色系**：靛蓝主色 + 状态色

| Token | 色值 | 用途 |
|-------|------|------|
| `--color-primary` | `#6366f1` (Indigo 500) | 主品牌色、主按钮、活跃状态、链接 |
| `--color-primary-light` | `#818cf8` (Indigo 400) | hover 状态、次要强调 |
| `--color-primary-bg` | `#eef2ff` (Indigo 50) | 卡片高亮背景、选中态底色 |
| `--color-success` | `#10b981` (Emerald 500) | 成功状态、完成标识 |
| `--color-danger` | `#ef4444` (Red 500) | 威胁情报、错误状态、警告 |
| `--color-warning` | `#f59e0b` (Amber 500) | 敏感度提醒、降级提示 |
| `--color-text-primary` | `#1e293b` (Slate 800) | 标题、正文 |
| `--color-text-secondary` | `#64748b` (Slate 500) | 描述文字、辅助信息 |
| `--color-text-muted` | `#94a3b8` (Slate 400) | placeholder、极次要信息 |
| `--color-border` | `#e2e8f0` (Slate 200) | 卡片边框、分割线 |
| `--color-surface` | `#ffffff` | 卡片背景 |
| `--color-background` | `#f8fafc` (Slate 50) | 页面底色 |

**使用规则**：

- 品牌色（靛蓝）只用于：主按钮、活跃导航项、进度条、关键链接。不要大面积铺色
- 状态色（绿/红/黄）只用于：对应语义的场景，不要跨语义使用
- 正文区域保持白底灰字，色彩克制

### 3.3 图标系统

**Phosphor Icons — Duotone 风格**

| 功能模块 | 图标名 | 用途 |
|---------|--------|------|
| 智能 Prep | `shield-check-duotone` | 安全准备、方案生成 |
| 模拟训练 | `chat-centered-dots-duotone` | 对话练习、角色扮演 |
| 知识库 | `database-duotone` | 知识检索、资料查阅 |
| 威胁情报 | `magnifying-glass-duotone` | 威胁搜索、情报分析 |

**嵌入方式**：

- Streamlit 中通过 `st.markdown` 嵌入 inline SVG
- 图标统一尺寸：24x24px（正文）、32x32px（卡片标题）、48x48px（Hero/入口卡片）
- 图标颜色跟随所在文本的 color 属性（`currentColor`）
- 禁止使用 emoji 替代图标，包括 `page_icon` 也换成 SVG favicon

### 3.4 排版规范

| 层级 | 字号 | 字重 | 颜色 | 用途 |
|------|------|------|------|------|
| H1 | 36px | 800 | `--color-text-primary` | 页面标题 |
| H2 | 24px | 700 | `--color-text-primary` | 分区标题 |
| H3 | 18px | 600 | `--color-text-primary` | 卡片标题 |
| Body | 15px | 400 | `--color-text-primary` | 正文内容 |
| Caption | 13px | 400 | `--color-text-secondary` | 辅助说明 |
| Small | 12px | 400 | `--color-text-muted` | 版本号、时间戳 |

- 行高：正文 1.6，标题 1.3
- 字体：系统默认（`-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`），不需要引入 Google Fonts
- 段落间距：正文段落间 16px，分区间 32px

### 3.5 间距系统

采用 8px 基准的间距阶梯：

| Token | 值 | 用途 |
|-------|-----|------|
| `--space-1` | 4px | 图标与文字间距 |
| `--space-2` | 8px | 紧凑元素内间距 |
| `--space-3` | 12px | 列表项间距 |
| `--space-4` | 16px | 卡片内间距、段落间距 |
| `--space-5` | 24px | 卡片 padding |
| `--space-6` | 32px | 分区间距 |
| `--space-8` | 48px | 页面顶部 Hero 间距 |

### 3.6 卡片规范

| 属性 | 值 |
|------|-----|
| 圆角 | 12px |
| 边框 | 1px solid `--color-border` |
| padding | 24px |
| 背景 | `--color-surface` |
| hover 边框色 | `--color-primary` |
| hover 阴影 | `0 4px 16px rgba(0,0,0,0.08)` |
| hover 位移 | `translateY(-2px)` |
| 过渡时间 | 200ms ease |
| cursor | pointer（可点击卡片） |

## 4. 页面规格

### 4.1 首页（Home.py）

**信息架构**：Hero → 4 个入口卡片（合并"核心功能"和"快速开始"）

**删除的内容**：

- "快速开始"三张大卡片（与"核心功能"信息重复）
- "技术架构" expander（对售前用户无价值，移至"关于"页面或侧边栏）
- 侧边栏的"快速导航"按钮（Streamlit 自动生成页面导航已覆盖）

**Hero 区域**：

- 标题："JARVIS"（36px, 800 字重），左侧放 Phosphor shield-check 图标（48px）
- 副标题："AI 驱动的销售拜访准备助手"（18px, `--color-text-secondary`）
- 描述语："融合行业知识库、威胁情报和 AI 洞察，为客户拜访做好充分准备"（14px, `--color-text-muted`）
- 居中对齐，上下 padding 各 48px

**入口卡片**（4 列网格）：

每张卡片结构：

```
┌─────────────────────────────┐
│  [Phosphor Icon 32px]       │
│                             │
│  标题（H3）                  │
│  描述文字（Caption, 2行）    │
│                             │
│  [进入 →] 文字链接           │
└─────────────────────────────┘
```

- 4 张卡片对应：智能 Prep / 模拟训练 / 知识库 / 威胁情报
- 点击卡片跳转到对应页面
- hover 时边框变品牌色 + 上浮 2px

**侧边栏**：

- 品牌区：JARVIS logo + 副标题
- 知识库概览：4 个 metric（案例/方法论/敏感度/产品）
- 底部：版本号 `JARVIS v0.3 · Prep Assistant`
- 侧边栏宽度：300px（CSS 强制），防止"Home"文字截断

### 4.2 智能 Prep 页面（1_智能Prep.py）

**信息架构**：输入区 → 生成进度 → 结果区（分区卡片）→ 导出工具栏

#### 4.2.1 输入区

**模板选择 + 自由输入**

模板卡片（4-5 个预设场景，横向排列）：

| 模板名 | 行业 | 预填文本 |
|--------|------|---------|
| 制造业勒索事件 | 制造业 | "明天拜访一家制造业客户，他们的产线刚遭受勒索软件攻击" |
| 金融合规检查 | 金融 | "拜访一家城商行客户，讨论等保 2.0 合规建设方案" |
| 医疗数据泄露 | 医疗 | "拜访一家三甲医院，他们近期发生了患者数据泄露事件" |
| 政府等保建设 | 政府 | "拜访某市教育局，讨论政务系统等保三级建设方案" |
| 自由输入 | — | （空文本框） |

- 用户点击模板卡片 → 自动填入文本框 → 高亮选中态
- 文本框在模板下方，支持自由编辑
- 行业标签从 3 个扩展为 7 个（覆盖知识库所有行业）

#### 4.2.2 生成过程

**步骤进度条**（使用 `st.status` 组件）：

```
✅ 识别行业与场景        制造业 · 勒索事件响应
✅ 检索知识库            匹配到 3 个案例、5 条敏感度规则
⏳ 生成 Prep 包          正在组织方案方向和话术要点...
⏸  拉取威胁情报          等待上一步完成
```

- 4 个步骤：意图识别 → 知识检索 → LLM 生成 → 情报拉取
- 每完成一步更新状态图标（✅）和摘要文案
- 降级到规则引擎时，在对应步骤显示警告色

#### 4.2.3 结果区

**分区卡片**（3 个分区，替代原来的 6 个 expander）：

**分区一：核心准备**（默认展开，直接可见）

包含：
- 场景判断（`pkg.scenario_assessment`）
- 方案方向（`pkg.solution_direction`）
- 话术要点（`pkg.talking_points`）

视觉：白色卡片，左侧 Phosphor `target-duotone` 图标，内容直接展示不折叠。

**分区二：参考素材**（默认折叠，可展开）

包含：
- 案例匹配（`pkg.matched_cases`）
- 追问清单（`pkg.follow_up_questions`）
- 敏感度提醒（`pkg.sensitivity_alerts`）

视觉：白色卡片，左侧 Phosphor `bookmark-simple-duotone` 图标，默认折叠，点击展开。

**分区三：威胁情报**（默认折叠，可展开）

包含：
- 威胁事件列表（`pkg.threat_intel`）

视觉：白色卡片，左侧 Phosphor `warning-diamond-duotone` 图标，默认折叠。如有威胁事件，卡片标题旁显示红色 badge 标注事件数量。

#### 4.2.4 导出工具栏

位于结果区右上角，包含：

| 按钮 | 功能 | 图标 |
|------|------|------|
| 导出 PPT 大纲 | 调用 `generate_outline(pkg)` 并下载 | `presentation-chart-duotone` |
| （预留）导出 Word | 未来功能 | `file-doc-duotone` |
| （预留）导出 PDF | 未来功能 | `file-pdf-duotone` |

- PPT 大纲点击后生成 → 显示预览 → 提供下载按钮
- 生成中显示 `st.spinner`

## 5. 交互规范

### 5.1 状态反馈

| 状态 | 视觉表现 |
|------|---------|
| 空闲 | 正常样式 |
| hover | 边框变品牌色 + 上浮 2px + 阴影加深 |
| 加载中 | 主按钮显示 spinner + 禁用，卡片显示骨架屏灰色占位 |
| 成功 | 品牌绿色边框 + 短暂绿色闪烁（200ms） |
| 错误 | 红色边框 + 错误文案显示在字段下方 |
| 降级 | 黄色边框 + "已切换为基础模式"提示 |

### 5.2 过渡动画

- 卡片 hover：`transition: all 200ms ease`
- 内容切换：`fade-in 300ms`
- 步骤进度：`width transition 400ms ease-out`
- 禁止使用超过 500ms 的动画

### 5.3 空状态

| 场景 | 空状态内容 |
|------|-----------|
| 知识库无匹配案例 | "未找到匹配案例。尝试换一个行业标签，或使用更通用的关键词描述场景。" |
| 威胁情报为空 | "当前行业暂无最新威胁事件。情报数据每小时更新一次。" |
| 生成失败 | "生成出错：[错误信息]。请检查 API Key 配置，或点击「重试」。" + 重试按钮 |

## 6. 技术约束

### 6.1 Streamlit 限制

- **不支持真正的组件库** — Streamlit 没有 component system，所有自定义 UI 通过 `st.markdown(html, unsafe_allow_html=True)` 实现
- **不支持客户端路由** — 页面切换靠 `st.switch_page()`，会触发全页刷新
- **不支持真正的流式渲染** — `st.status` / `st.progress` 是同步更新，不是 WebSocket 推送
- **CSS 作用域** — Streamlit 的 CSS 是全局的，需要通过 `data-testid` 选择器精确定位

### 6.2 兼容性

- 目标 Streamlit 版本：>= 1.26（需要 `st.status` 组件）
- 浏览器：Chrome 90+ / Edge 90+ / Safari 15+
- 不需要移动端适配（售前场景以桌面端为主）

## 7. 验收标准

### 7.1 首页

- [ ] "Home" 在侧边栏中完整显示，不截断
- [ ] 侧边栏无重复导航（只有 Streamlit 自动生成的 + 知识库概览）
- [ ] 4 个入口卡片使用 Phosphor SVG 图标，无 emoji
- [ ] 卡片 hover 有边框变色 + 上浮动效
- [ ] 页面只有一屏内容，不需要滚动

### 7.2 智能 Prep

- [ ] 输入区有 4-5 个场景模板卡片可点击
- [ ] 生成过程显示 4 步进度条，每步有状态文案
- [ ] 结果分为 3 个分区卡片（核心准备/参考素材/威胁情报）
- [ ] 核心准备区默认展开，内容直接可见
- [ ] 参考素材和威胁情报默认折叠
- [ ] 导出工具栏在结果区右上角，PPT 大纲功能可用
- [ ] 降级到规则引擎时，进度条对应步骤显示黄色警告
- [ ] 全页面无 emoji 作为功能图标

### 7.3 通用

- [ ] 所有页面使用统一的色彩 token（无硬编码色值）
- [ ] 所有图标为 Phosphor duotone SVG（无 emoji）
- [ ] 间距遵循 8px 基准阶梯
- [ ] 卡片统一圆角 12px、border 1px、padding 24px
- [ ] 过渡动画不超过 300ms
